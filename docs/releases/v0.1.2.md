# Daylo v0.1.2

Daylo v0.1.2 removes the user-facing synchronization step. Reading weight data now retrieves
newly available measurements automatically before returning the normalized result.

## Changes

- Make `daylo latest` and `daylo list` retrieve newly available measurements automatically.
- Remove the `daylo sync` command and `POST /api/v1/sync` endpoint.
- Coalesce concurrent reads so they do not issue duplicate provider requests.
- Reuse a successful Tanita retrieval for up to 60 seconds to respect provider limits.
- Return the most recently stored measurements when a provider is temporarily unavailable.
- Keep the existing `latest` and `list` response shapes unchanged.

## Install

```sh
bunx daylo-cli@0.1.2 --help
```

## Supported Providers

- Withings
- Tanita Health Planet

## Known Limits

- Provider coverage is limited to Withings and Tanita Health Planet.
- Automatic retrieval happens when weight data is read; this release does not add webhooks or
  scheduled background retrieval.
- The public repository contains the CLI, API contract, documentation, and agent-facing usage
  files. The hosted backend remains part of the Daylo service.
